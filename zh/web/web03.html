1 . 请说一说二次开发的优缺点? #

  • 优点:
    • 基于成熟的架子开发, 后期项目的扩展性和可维护性非常强 (市场普遍采用)
    • 有很成熟的各种功能模块, 可以快速定制开发
  • 缺点:
    • 因为作为通用型的架子, 要考虑到各种开发需求, 架子的模块粒度非常细, 组件封装度非常高
    • 对刚开始接触的小伙伴不太友好, 太多的文件夹, 太多的组件层层引入 (但一切都是为了可扩展)
    • 但是, 一旦熟悉和掌握了内在思路, 对咱们的成长帮助非常大

2. 请描述下RBAC? #

  • role based access control 基于角色的权限控制
  • 给员工分配角色 给角色分配对应的权限

3. 请说一说项目中使用图标的几种方案? #

  • 精灵图 优点: 多个小图片放在一张大图片中, 减少了请求次数 缺点: 图片会失真, 使用时获取位置和尺寸比较麻烦(得借助工具)

  • 字体图标 优点: 图标不会失真 缺点: 每次新增图标, 需要去站点配置, 需要重新更新字体文件

  • 改进方案: svg精灵图 svg图标: 不失真, 使用方便 svg精灵图: 把多个svg图标放在一起, 使用的时候直接根据id获取

6. 请说一说native修饰符? #

  • 正常情况下, 往一些自定义组件身上去加原生事件是不生效的, 比如: click, keyup...

  • 如果让自定义组件支持原生事件呢?

    • 自定义事件
    • 加.native修饰符

  • 加.native修饰符底层逻辑是:

​ vue一旦发现自定义组件注册了原生事件, 同时又加了.native修饰符

​ 它就会把原生事件注册在组件内部的根元素上去

  • 结论:

​ 以后, 凡是往自定义组件上加原生事件不生效, 直接加.native修饰即可

       ]
   },

/**
 * 把 列表数据数组 转成  树形数据数组
 * @param {*} listArr 列表数据数组
 * @param {*} flag  转换标记
 */
export const transListToTreeArr = (listArr, flag) => {
  // 1. 定义一个数组
  const treeArr = []
  // 2. 遍历列表数组
  listArr.forEach((item) => {
    // 2.1 判断
    if (item.pid === flag) {
      const resArr = transListToTreeArr(listArr, item.id)
      if (resArr.length > 0) {
        item.children = resArr
      }
      treeArr.push(item)
    }
  })

  // 3. 返回树形数组
  return treeArr
}

10-配置代理解决跨域 #

【面试提问】：简单聊聊对于跨域 和 跨域解决方案的理解?

【简历体现】：理解跨域的原理, 并能够独立配置服务器代理解决跨域

【答题大纲】：

  1. 什么是跨域？
  2. 跨域的解决方案有哪些？
  3. 如何配置的？

具体话术：

跨域问题 其实是 浏览器 的安全策略问题，浏览器有一个同源策略，必须域名端口和协议都相同，才会正常接受请求结果，只要有一个不同，就会认为跨域了，数据不安全，不予接受。

跨域的解决方案具体有以下三种：jsonp cors 代理

  1. jsonp用的不太多了，优点是兼容IE6，但是缺点是只支持get请求，底层是动态创建script标签，利用src进行脚本的加载，进行接受数据

  2. cors用的还挺多，因为方便，只需要后台需要配置响应头，前端正常请求即可，缺点就是有一定兼容性，需要IE10+

  3. 代理用的最多，兼容性也比较好，就是需要一定的配置。我们公司用的一般是代理解决跨域，开发环境webpack配置代理，生产环境Nginx配置代理

怎么配置的webpack代理？就是vue.config.js配置proxy配置项，然后配置路径别名和需要映射的地址

11-登录token相关 #

【面试提问】：登录模块怎么做的？聊聊token的处理？

【简历体现】：能够熟练完成对于token的vuex存储和持久化，实现登录模块

【答题大纲】:

  • 请求获取token
  • vuex存token
  • storage持久化
  • 登录拦截
  • token过期

具体话术：

首先我们，在登录页，用户点击登录按钮，需要发送请求获取token

拿到token，需要存入到vuex中，可以方便的所有组件中获取，且能响应式更新

但是vuex刷新又会丢失，所以我们也会存一份到本地，一般可以通过localStorage 或者 cookie

而且我们的系统内部网页，未登录的用户，是不让访问的，所以我们这边还加了登录拦截。【过渡话术】

登录拦截如何实现的，就是在路由前置导航守卫的地方，判断token是否存在，判断是否再白名单中，

如果token存在，或者是白名单中直接放行，否则拦截到登录。

最后，token也是会过期的，如果token过期了，后台会返回401状态码，需要统一在响应拦截器处理。

散的点：

  1. vuex具体怎么存？

    第一步，页面中 dispatch 一个 action

    第二步，action中发送请求，获取token

    第三步，action中提交mutation，将token提交

    第四部，mutation中将token存入vuex

  2. 为什么要存vuex？

    像个人信息，token信息等，都是放vuex比较【方便】

    便于所有的组件，都能快速获取当前用户的权限信息，且能响应式变化，比较【灵活】

15-路由设计 #

【面试提问】：您的公司中路由是怎么设计的?

【简历体现】：独立根据项目需求, 拆分路由模块, 设计静态路由和动态路由, 便于动态进行权限控制

【答题大纲】：

  1. 拆分路由模块的理解: 为啥要拆分路由模块?
  2. 静态路由 动态路由的理解?
  3. 如何进行权限控制?

具体话术：

  1. 根据不同的模块， 拆分路由 （为啥我要拆分？ 直接写在一起不行？） 我们公司模块特别多，光模块就有二十多个，我负责四个大的模块 如果所有的路由全都写在一起，一来容易造成代码冲突，二来也不易维护，三来不易于权限控制

  2. 静态路由 和 动态路由

    • 静态路由：一上来就都开放给所有登录用户的路由，定死分配的一些路由规则 不会随着角色身份权限的变化而变化
    • 动态路由：登录后，在导航守卫处获取用户的个人信息（包含权限）， 比如按钮权限，菜单权限，接口访问权限等，其中菜单权限最重要，用于动态分配控制路由 如果用户有这个菜单权限，那么就给他加上这个路由，有十个导航菜单 如果用户没有这个菜单权限，那么就不加上这个路由，就没有这些菜单按钮

    其实我们公司，菜单导航，也完全是基于路由，动态生成的

16-防抖和节流 #

【面试提问】：简单聊聊防抖和节流的理解?

【简历体现】：能够熟练使用 防抖 和 节流 实现项目功能 性能优化

【答题大纲】:

  1. 防抖和节流存在的意义
  2. 防抖和节流的概念和使用场景

具体话术：

防抖 和 节流存在的意义: 都是为了性能优化, 可以阻止事件的频繁触发

节流: <立刻>执行某件事, 但是事件执行的过程中, 关闭<节流阀>, 只有当事件执行完, 才能执行第二次

场景:

  • (1) 点按钮loading请求, 发请求立刻发, 发的过程中, 不会发第二次
  • (2) 轮播图, 点击箭头, 开始滑动, 滑动的过程中, 关闭节流阀, 等动画结束, 才能执行第二次
  • (3) 下拉刷新, 上拉加载更多, 一旦请求了, loading状态, 关闭节流阀

防抖: <延迟>执行某件事, 会开个延时器, 等一会儿, 如果事件再次触发, 重开延时器, 直到一段时间内, 都没有触发了, 才执行

场景:

  • (1) resize事件, 屏幕修改宽度高度, 会非常频繁的触发 一般需求是, 等用户修改完屏幕大小后, 做适配 => 延迟执行
  • (2) 输入框搜索提示, 实时每个字母都立刻做提示, 频繁触发了 一般需求是, 等用户一段时间没有输入新的内容, 做搜索提示 => 延迟执行

  2. 要有出口，要有结束条件，否则死递归 出口：有没有子代children的存在，没有就结束了

  <div v-if="item.children && item.children.length > 0" > 
     <hm-tree :data="item.children"/>
  </div>

  1. 树形组件内部的结构需要定制 插槽: 默认插槽(具名) 作用域插槽 (1) 如果没有传递插槽内容，使用插槽后备内容， 将节点展示 (2) 传递了插槽内容，需要在递归组件内部，将一层插槽内容，往下传递，传递给二层插槽 否则二层插槽，只会渲染默认节点
    hm-tree.vue

     <div>
     <slot name="default">后备内容</slot>
     <hm-tree>
         <template #default>
             <slot name="default"></slot>  // 将一层拿到的结构，往下传递给二层
         </template>
     </hm-tree>
 </div>

    (3) 插槽的使用结构定制时，需要参数的，就需要将有的参数，配置给slot，通过<作用域插槽的方式传值>
    hm-tree.vue

    <div v-for="xxx">
    <slot name="default" :label="item.label" :index="index" :treeData="data"></slot>
    <hm-tree>
        <template #default="obj">
            <slot name="default" v-bind="obj"></slot>  
        </template>
    </hm-tree>
</div>

v-bind="obj" 将二层三层插槽中绑定的数据拿到，传递给渲染时的结构层

树形数据的处理话术：

  1. 首先后台返回的数据，是列表式的数据结构
  2. 需求：将列表式的数据转换成树形结构
  3. 关键字段：id pid 根据 pid 确定父级
  4. 实现：遍历先找一级，确定一级后，根据一级的id作为下一层的pid，查询渲染二级，并递归

面试追问：你这样处理，数据量一旦大了，不是会很卡么？

我们公司有两个业务需求情况：

  1. 数据量比较简单，部门列表结构展示，直接一次性处理完
  2. 而有些数据量比较大的情况，分批次渲染 为了性能考虑，不会一次性全部树形数据解析完，效率太低 做法：
    1. 只显示一级
    2. 当点击一级时时，再加载二级
    3. 可以添加loading效果，增加用户体验 （element tree 支持loading加载）

            username: 'xx',
            password: 'xx'
         }
       })
     特点：
       内存中传递，刷新会丢失
     如何获取  this.$route.params.username

27-vue的原理知道么 #

【概念1】：MVVM

  • vue是一个MVVM的框架，核心思想就是数据驱动视图
    • M就是model，指的是数据
    • V就是view，指的是视图
    • VM就是viewmodel，是一个桥梁，可以同时操作视图和数据
  • 我们只需要修改数据，视图会自动更新

【概念2】：数据劫持

  • 那么vue底层是如何监听数据的变化的呢？如何进行的数据劫持呢？
    • vue2 中用的是 Object.defineProperty
    • vue3 中用的是 proxy
  • 监听到数据变化后，动态的重新生成虚拟dom，对比新旧dom差异，动态的异步更新视图

【概念3】：虚拟dom

  • 什么是虚拟dom呢，虚拟dom就是一个个 js 对象，用来描述真实dom的，
  • 可以比真实dom更简洁，提高对比性能

【概念4】：如何对比 diff算法

  • 怎么对比的呢？底层是diff算法

    • 优先进行根级别的元素比较，如果根级别元素一样，可以复用，可以继续进行比对，

      如果根级别不一样，直接销毁重建

    • 其次比较同级兄弟元素，兄弟元素默认按照下标顺序比较

      如果设置了key唯一标识，按照相同key的比较

29-图片懒加载 #

【面试提问】简单聊聊图片懒加载？？

关键词：监听页面滚动 判断图片是否在可视区 动态设置src

拆解：

  1. 是什么？

    图片懒加载：就是图片一开始没在可视区时，不加载，进入到可视区才加载。

  2. 基本原理？原生

    (1) 图片一上来不设置 src，如果一上来设置了就会立刻发请求 或者一上来设置一张默认图片 (2) 当图片将进入到可视区范围内，才加载设置 src 那么如何判断进入可视区范围的呢？

       1. 监听页面的滚动
   2. 获取盒子距离顶部的距离
   3. 一旦盒子距离顶部的距离 < (可视区高度 + 页面卷去高度)  说明进入可视区了
   4. 设置图片的src即可

  3. 工作应用

    组件库一般都会内置, 无论 vant 还是 element-ui， 注意：懒加载的容器都得设置宽高

    element el-image组件 lazy属性 实现懒加载

    vant van-image组件 lazy-load属性 实现懒加载

  4. 有什么好处？

    优化性能，减轻服务器压力

  • 使用场景

  • 扩展
