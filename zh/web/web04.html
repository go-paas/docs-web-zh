Vue基础 #

第一天

1. vue是什么? 什么是渐进式呢? 什么是库和框架呢? #

  • vue是一个渐进式的javascript框架

  • 逐渐增强，vue不强求我们一次性在网站中运用学习所有的语法，可以学一点用一点

  • 库：本质上是一些方法的集合; 每次调用方法，实现一个特定的功能。比如: 如：moment axios

  • 框架：是一套完整的解决方案。框架实现了大部分的功能，我们需要按照框架的规则写代码。比如: vue react angular ...

2. 什么是 MVVM ？M、V、VM分别是什么？ #

  • MVVM: Model-View-ViewModel, 一种软件架构模式; Vue就是一个MVVM的框架
  • Model：模型层，负责处理业务逻辑以及和服务器端进行交互
  • View：视图层：负责将数据模型转化为UI展示出来，可以简单的理解为HTML页面
  • ViewModel：视图模型层，用来连接Model和View，是Model和View之间的通信桥梁

3. 原生 和 vue 修改视图的思想分别是什么？ #

  • 原生：dom驱动，修改视图，操作dom
  • vue：数据驱动，修改数据，操作数据即可
  • Vue所有的界面事件，都是只去操作数据的，原生JS则是操作DOM
  • Vue所有界面的变动，都是根据数据自动绑定出来的，原生JS还是操作DOM

4. 什么是组件化？组件化的好处是什么？ #

  • 将一个页面，拆分成一个个小组件的过程，就是组件化。每个组件包含自己的 html+css+js
  • 在Vue中每一个.vue文件都可以视为一个组件
  • 组件化的优势:
    • 便于维护, 由于每个组件的职责单一, 降低整个系统的耦合度
    • 便于复用, 在保持接口不变的情况下，我们可以替换不同的组件快速完成需求
    • 调试方便，由于整个系统是通过组件组合起来的，在出现问题的时候，可以用排除法直接移除组件，或者根据报错的组件快速定位问题，之所以能够快速定位，是因为每个组件之间低耦合，职责单一，所以逻辑会比分析整个系统要简单

5. 什么是Vue脚手架? 它的好处是什么? #

  • @vue/cli 也叫 vue脚手架, @vue/cli 是vue官方提供的一个全局命令工具
  • 这个命令可以帮助我们快速的创建一个vue项目的基础架子, 大大提高了我们开发项目的效率
  • 脚手架的作用:
    • 开箱即用
    • 零配置（不用你配webpack）
    • 内置babel等工具

第二天

6. vue指令-v-on的三种语法? 并说出自己的理解? #

  • v-on:事件名=“要执行的少量代码"
    • 当逻辑只有一句代码, 称之为少量代码, 可以把代码直接写在页面里面
  • v-on:事件名=“methods中的函数名"
    • 当逻辑比较复杂时, 可以把逻辑代码写在函数中, 此处调用函数即可
  • v-on:事件名=“methods中的函数名(实参)"
    • 当逻辑比较复杂时, 而且函数中的代码需要根据传递的参数来分别实现

7. vue中如何获取事件对象呢？ #

  • 没有传参, 通过形参接收e

  • 传参了, 通过$event接收事件对象e

    • $event 一个固定名字的形参

    • 一般$event放在参数的末尾

8. 请说说v-if和v-show的区别? #

共同点:

  • 在 vue 中 v-show 与 v-if 的作用效果是相同的(不含v-else情况)，都能控制元素在页面是否显示

  • 在用法上也是相同的

    // 当isShow为true的时候，都会占据页面的位置
// 当isShow为false时，都不会占据页面位置
<div v-show="isShow" />
<div v-if="isShow" />

不同点:

  • 控制手段不同

    • v-show隐藏则是为该元素添加css--display:none，dom元素依旧还在

    • v-if显示隐藏是将dom元素整个添加或删除

  • 性能消耗不同

    • v-if有更高的切换消耗
    • v-show有更高的初始渲染消耗

应用场景:

  • v-if 与 v-show 都能控制dom元素在页面的显示
  • v-if 相比 v-show 开销更大的（直接操作dom节点增加与删除）
  • 如果需要非常频繁地切换，则使用 v-show 较好
  • 如果在运行时条件很少改变，则使用 v-if 较好

9. Vue常用的修饰符有哪些? 有什么应用场景? #

  • 修饰符是什么?

    • 在Vue中，修饰符处理了许多DOM事件的细节，让我们不再需要花大量的时间去处理这些烦恼的事情，而能有更多的精力专注于程序的逻辑处理
    • vue中常用的修饰符有:
      • 表单修饰符
      • 事件修饰符
      • 键值修饰符

  • 修饰符的作用?

    1. 表单的修饰符有如下:

    • lazy 在我们填完信息，光标离开标签的时候，才会将值赋予给value，也就是在change事件之后再进行信息同步

    • trim 自动过滤用户输入的首空格字符，而中间的空格不会过滤

    • number 自动将用户的输入值转为数值类型，但如果这个值无法被parseFloat解析，则会返回原来的值

    1. 事件修饰符是对事件捕获以及目标进行了处理，有如下常用修饰符:

    • stop 阻止了事件冒泡，相当于调用了event.stopPropagation方法

    • prevent 阻止了事件的默认行为，相当于调用了event.preventDefault方法

    1. 键盘修饰符是用来修饰键盘事件（@keyup，@keydown）的，有如下:

    enter、tab、delete、space、esc、up...

10. 什么是虚拟DOM? #

虚拟DOM只是一层对真实DOM的抽象，以JavaScript 对象 (VNode 节点) 作为基础的树，用对象的属性来描述节点，最终可以通过一系列操作使这棵树映射到真实环境上

在Javascript对象中，虚拟DOM 表现为一个 Object对象。并且最少包含标签名 (tag)、属性 (attrs) 和子元素对象 (children) 三个属性

真实DOM:

<div id="app">
    <p class="p">节点内容</p>
</div>

虚拟DOM:

const dom = {
    type: 'div',
    attributes: [{id: 'box'}],
    children: [
        {
            type: 'p',
            attributes: [{class: 'p'}],
            text: '节点内容'
        }
    ]
}

通过虚拟DOM，vue可以对这颗抽象树进行创建节点, 删除节点以及修改节点的操作， 经过diff算法得出一些需要修改的最小单位, 再更新视图，减少了dom操作，提高了性能

很多人认为虚拟 DOM 最大的优势是 diff 算法，减少 JavaScript 操作真实 DOM 的带来的性能消耗。

虽然这一个虚拟 DOM 带来的一个优势，但并不是全部。

虚拟 DOM 最大的优势在于抽象了原本的渲染过程，实现了跨平台的能力，而不仅仅局限于浏览器的 DOM，可以是安卓和 IOS 的原生组件，可以是近期很火热的小程序，也可以是各种GUI

第三天

11. 请说一说Vue中就地复用策略? #

  • Vue会尽可能的就地（同层级，同位置），对比虚拟dom，复用旧dom结构，进行差异化更新。

  • 就地复用的好处在于可以复用旧的dom结构，更新高效！

12. 请说说什么是虚拟dom? 为啥不对比真实dom呢？ #

  • 虚拟dom：本质就是保存节点信息, 描述真实dom的 JS对象
  • 真实dom太复杂，对比性能效率低, 所以要对比虚拟dom

13. 请描述下diff算法的对比策略? #

  • 策略1：先同层级根元素比较

    • 如果根元素变化，那么不考虑复用，整个dom树删除重建
    • 如果根元素不变，对比出属性的变化更新，并考虑往下递归复用。

  • 策略2：对比同级兄弟元素时，默认按照下标进行对比复用。

    • 如果指定了key，就会按照相同key的元素来进行对比复用

14. Vue为什么要用虚拟Dom? #

  • 虚拟dom就是用js对象来描述真实Dom，是对真实Dom的抽象
  • 由于直接操作Dom性能低，但是js层的操作效率高，可以将Dom操作转化成对象操作。最终通过diff算法比对差异进行更新Dom
  • 虚拟Dom不依赖真实平台环境，可以实现跨平台

15. 请说说v-html的底层实现, 使用时会导致哪些问题? #

  • 核心点: innerHTML
  • XSS 攻击
  • v-html 会替换标签内部的元素

第四天

16. 请说说computed watch 和 methods? #

computed：默认computed也是一个watcher具备缓存，只有当依赖的数据变化时才会计算, 当数据没有变化时, 它会读取缓存数据。如果一个数据依赖于其他数据，使用 computed

watch：每次都需要执行函数。 watch 更适用于数据变化时的异步操作。如果需要在某个数据变化时做一些事情，使用watch。

methods：只要把方法用到模板上了,每次一变化就会重新渲染视图，性能开销大

17. 谈谈你对组件的理解？ #

  • 组件化开发能大幅提高应用开发效率、测试性、复用性
  • 常用的组件化技术：属性、自定义事件、插槽
  • 降低更新范围，值重新渲染变化的组件
  • 高内聚、低耦合、单向数据流

18. v-for 为什么要加 key? #

  • 如果不使用 key，Vue 会最大限度使用就地复用策略, 如果对内容进行更改会造成数据混乱

  • key 是为 Vue 中 vnode 的唯一标记，通过这个 key，我们的 diff 操作可以更准确、更快速

    • 更准确：因为带 key 就不是就地复用了，在 sameNode 函数 a.key === b.key 对比中可以避免就地复用的情况, 所以会更加准确。

    • 更快速：利用 key 的唯一性生成 map 对象来获取对应节点，比遍历方式更快

19. 说一下v-model的原理? #

  • v-model本质就是一个语法糖，可以看成是value + input方法的语法糖。

  • 原生的v-model，会根据标签的不同生成不同的事件和属性。

  • 可以通过model属性的prop和event属性来进行自定义。

  • 双向绑定的原理就是利用@input和:value来实现的

20. 组件写name有啥好处？ #

  • 增加name属性，会在components属性中增加组件本身，实现组件的递归调用。
  • 可以表示组件的具体名称，方便调试和查找对应的组件。

24. 请说说单向数据流? #

  • 在vue中需要遵循单向数据流原则: 从父到子的单向数据流动, 叫单向数据流

  • 规律

    • 父组件的数据变化了，会自动向下流动影响到子组件

    • 子组件不能直接修改父组件传递过来的 props， props是只读的！

25. 子传父的基本步骤是什么? #

  • 子组件内, 恰当时机 this.$emit('自定义事件名', 值)
  • 父组件内, 给组件 @自定义事件="父methods函数名"

26. 在 Vue中 如何获取真实DOM元素呢 ? #

27. Vue中的$nextTick有什么作用？ #

  • 官方对其的定义
    • 在下次 DOM 更新循环结束之后执行延迟回调。在修改数据之后立即使用这个方法，获取更新后的 DOM
  • 什么意思呢？
    • 我们可以理解成，Vue 在更新 DOM 时是异步执行的。
    • 当数据发生变化，Vue将开启一个异步更新队列，视图需要等队列中所有数据变化完成之后，再统一进行更新
  • 使用场景?
    • 如果想要在修改数据后立刻得到更新后的DOM结构，可以使用Vue.nextTick()

28. 你有写过自定义指令吗？自定义指令的应用场景有哪些？ #

  • 注册一个自定义指令有: 全局注册与局部注册

  • 全局注册主要是通过Vue.directive方法进行注册

    • Vue.directive第一个参数是指令的名字（不需要写上v-前缀），第二个参数可以是对象数据，也可以是一个指令函数

      // 注册一个全局自定义指令 `v-focus`
Vue.directive('focus', {
  // 当被绑定的元素插入到 DOM 中时……
  inserted: function (el) {
    // 聚焦元素
    el.focus()  // 页面加载完成之后自动让输入框获取到焦点的小功能
  }
})

  • 局部注册通过在组件options选项中设置directive属性

    directives: {
  focus: {
    // 指令的定义
    inserted: function (el) {
      el.focus() // 页面加载完成之后自动让输入框获取到焦点的小功能
    }
  }
}

  • 然后你可以在模板中任何元素上使用新的 v-focus property，如下：

<input v-focus />

  • 应用场景

    • 图片懒加载

    • 页面水印

    • 权限校验

    • ...

29. Vue组件之间的通信方式都有哪些？ #

  • 组件之间的通信分类:

    • 父子组件之间的通信
    • 兄弟组件之间的通信
    • 祖孙与后代组件之间的通信
    • 非关系组件间之间的通信

  • vue中8种常规的通信方案:

    1. 通过 props 传递 √
    2. 通过 $emit 触发自定义事件 √
    3. 使用 ref
    4. EventBus
    5. $parent 或$root
    6. attrs 与 listeners
    7. Provide 与 Inject
    8. Vuex

  • props传递数据

    • 适用场景：父组件传递数据给子组件
    • 子组件设置props属性，定义接收父组件传递过来的参数
    • 父组件在使用子组件标签中通过字面量来传递值

  • $emit 触发自定义事件

    • 适用场景：子组件传递数据给父组件

    • 子组件通过$emit触发自定义事件，$emit第二个参数为传递的数值

    • 父组件绑定监听器获取到子组件传递过来的参数

30. 说说你对slot的理解？slot使用场景有哪些？ #

  • slot 是什么?

    Slot 艺名插槽，花名“占坑”，我们可以理解为solt在组件模板中占好了位置，当使用该组件标签时候，组件标签里面的内容就会自动填坑（替换组件模板中slot位置），作为承载分发内容的出口

    可以将其类比为插卡式的FC游戏机，游戏机暴露卡槽（插槽）让用户插入不同的游戏磁条（自定义内容）

  • 使用场景

    • 通过插槽可以让用户可以拓展组件，去更好地复用组件和对其做定制化处理
    • 如果父组件在使用到一个复用组件的时候，获取这个组件在不同的地方有少量的更改，如果去重写组件是一件不明智的事情
    • 通过slot插槽向组件内部指定位置传递内容，完成这个复用组件在不同场景的应用
    • 比如布局组件、表格列、下拉选、弹框显示内容等

  • 分类

    • 默认插槽
    • 具名插槽
    • 作用域插槽

第七天

31. 什么是vue的生命周期函数？整体生命周期分为几大阶段？ #

  • 生命周期函数

32. 请简单介绍下vue中生命周期函数? #

  1. 初始化阶段
  • beforeCreate：data数据初始化之前，组件还没有数据
  • created: data数据初始化之后，可以获取到组件的数据
    • 可以发送数据请求了
  • beforeMount：DOM渲染之前，DOM还没渲染
  • mounted：DOM渲染之后，可以操作DOM了
    • 可以操作DOM元素了
  1. 运行阶段
  • beforeUpdate: 数据更新，DOM更新前
  • updated: 数据更新，DOM更新后
  1. 销毁阶段
  • beforeDestroy: 组件销毁前
  • destroyed: 组件销毁后
    • 用来关闭定时器延时器、释放服务器资源等

vue-router与location.href的用法区别 #

  • vue-router使用pushState进行路由更新，静态跳转，页面不会重新加载；location.href会触发浏览器，页面重新加载一次
  • vue-router使用diff算法，实现按需加载，减少dom操作
  • vue-router是路由跳转或同一个页面跳转；location.href是不同页面间跳转；
  • vue-router是异步加载this.$nextTick(()=>{获取url})；location.href是同步加载

/**
 * 把 列表数据数组 转成  树形数据数组
 * @param {*} listArr 列表数据数组
 * @param {*} flag  转换标记
 */
export const transListToTreeArr = (listArr, flag) => {
  // 1. 定义一个数组
  const treeArr = []
  // 2. 遍历列表数组
  listArr.forEach((item) => {
    // 2.1 判断
    if (item.pid === flag) {
      const resArr = transListToTreeArr(listArr, item.id)
      if (resArr.length > 0) {
        item.children = resArr
      }
      treeArr.push(item)
    }
  })

  // 3. 返回树形数组
  return treeArr
}

10-配置代理解决跨域 #

【面试提问】：简单聊聊对于跨域 和 跨域解决方案的理解?

【简历体现】：理解跨域的原理, 并能够独立配置服务器代理解决跨域

【答题大纲】：

  1. 什么是跨域？
  2. 跨域的解决方案有哪些？
  3. 如何配置的？

具体话术：

跨域问题 其实是 浏览器 的安全策略问题，浏览器有一个同源策略，必须域名端口和协议都相同，才会正常接受请求结果，只要有一个不同，就会认为跨域了，数据不安全，不予接受。

跨域的解决方案具体有以下三种：jsonp cors 代理

  1. jsonp用的不太多了，优点是兼容IE6，但是缺点是只支持get请求，底层是动态创建script标签，利用src进行脚本的加载，进行接受数据

  2. cors用的还挺多，因为方便，只需要后台需要配置响应头，前端正常请求即可，缺点就是有一定兼容性，需要IE10+

  3. 代理用的最多，兼容性也比较好，就是需要一定的配置。我们公司用的一般是代理解决跨域，开发环境webpack配置代理，生产环境Nginx配置代理

怎么配置的webpack代理？就是vue.config.js配置proxy配置项，然后配置路径别名和需要映射的地址

首先我们，在登录页，用户点击登录按钮，需要发送请求获取token

拿到token，需要存入到vuex中，可以方便的所有组件中获取，且能响应式更新

但是vuex刷新又会丢失，所以我们也会存一份到本地，一般可以通过localStorage 或者 cookie

而且我们的系统内部网页，未登录的用户，是不让访问的，所以我们这边还加了登录拦截。【过渡话术】

登录拦截如何实现的，就是在路由前置导航守卫的地方，判断token是否存在，判断是否再白名单中，

如果token存在，或者是白名单中直接放行，否则拦截到登录。

最后，token也是会过期的，如果token过期了，后台会返回401状态码，需要统一在响应拦截器处理。

散的点：

  1. vuex具体怎么存？

    第一步，页面中 dispatch 一个 action

    第二步，action中发送请求，获取token

    第三步，action中提交mutation，将token提交

    第四部，mutation中将token存入vuex

  2. 为什么要存vuex？

    像个人信息，token信息等，都是放vuex比较【方便】

    便于所有的组件，都能快速获取当前用户的权限信息，且能响应式变化，比较【灵活】

16-防抖和节流 #

【面试提问】：简单聊聊防抖和节流的理解?

【简历体现】：能够熟练使用 防抖 和 节流 实现项目功能 性能优化

【答题大纲】:

  1. 防抖和节流存在的意义
  2. 防抖和节流的概念和使用场景

具体话术：

防抖 和 节流存在的意义: 都是为了性能优化, 可以阻止事件的频繁触发

节流: <立刻>执行某件事, 但是事件执行的过程中, 关闭<节流阀>, 只有当事件执行完, 才能执行第二次

场景:

  • (1) 点按钮loading请求, 发请求立刻发, 发的过程中, 不会发第二次
  • (2) 轮播图, 点击箭头, 开始滑动, 滑动的过程中, 关闭节流阀, 等动画结束, 才能执行第二次
  • (3) 下拉刷新, 上拉加载更多, 一旦请求了, loading状态, 关闭节流阀

防抖: <延迟>执行某件事, 会开个延时器, 等一会儿, 如果事件再次触发, 重开延时器, 直到一段时间内, 都没有触发了, 才执行

场景:

  • (1) resize事件, 屏幕修改宽度高度, 会非常频繁的触发 一般需求是, 等用户修改完屏幕大小后, 做适配 => 延迟执行
  • (2) 输入框搜索提示, 实时每个字母都立刻做提示, 频繁触发了 一般需求是, 等用户一段时间没有输入新的内容, 做搜索提示 => 延迟执行

    • 部门编码 不允许重复 （判断和所有部门的code是否重复）

19-树形组件封装 #

【面试提问】有没有封装过树形组件？比如树形菜单，比如树形选项卡之类的？

【简历体现】能够应用递归组件，封装树形结构组件，完成业务需求

核心要点：递归组件 树形数据

递归组件话术：

  1. 组件内部自己调用自己，必须起组件名name，才能调用到自己 hm-tree.vue

      <div>
     <div>内容说明</div>
     <hm-tree></hm-tree>
  </div>

  2. 要有出口，要有结束条件，否则死递归 出口：有没有子代children的存在，没有就结束了

   <div v-if="item.children && item.children.length > 0" > 
      <hm-tree :data="item.children"/>
   </div>

  1. 树形组件内部的结构需要定制 插槽: 默认插槽(具名) 作用域插槽 (1) 如果没有传递插槽内容，使用插槽后备内容， 将节点展示 (2) 传递了插槽内容，需要在递归组件内部，将一层插槽内容，往下传递，传递给二层插槽 否则二层插槽，只会渲染默认节点 hm-tree.vue

     <div>
     <slot name="default">后备内容</slot>
     <hm-tree>
         <template #default>
             <slot name="default"></slot>  // 将一层拿到的结构，往下传递给二层
         </template>
     </hm-tree>
 </div>

    (3) 插槽的使用结构定制时，需要参数的，就需要将有的参数，配置给slot，通过<作用域插槽的方式传值> hm-tree.vue

    <div v-for="xxx">
    <slot name="default" :label="item.label" :index="index" :treeData="data"></slot>
    <hm-tree>
        <template #default="obj">
            <slot name="default" v-bind="obj"></slot>  
        </template>
    </hm-tree>
</div>

v-bind="obj" 将二层三层插槽中绑定的数据拿到，传递给渲染时的结构层

树形数据的处理话术：

  1. 首先后台返回的数据，是列表式的数据结构
  2. 需求：将列表式的数据转换成树形结构
  3. 关键字段：id pid 根据 pid 确定父级
  4. 实现：遍历先找一级，确定一级后，根据一级的id作为下一层的pid，查询渲染二级，并递归

面试追问：你这样处理，数据量一旦大了，不是会很卡么？

我们公司有两个业务需求情况：

  1. 数据量比较简单，部门列表结构展示，直接一次性处理完
  2. 而有些数据量比较大的情况，分批次渲染 为了性能考虑，不会一次性全部树形数据解析完，效率太低 做法：
    1. 只显示一级
    2. 当点击一级时时，再加载二级
    3. 可以添加loading效果，增加用户体验 （element tree 支持loading加载）

20-sync修饰符 和 v-model #

【面试提问】： 封装组件时有没有用过 v-model 或者 .sync 修饰符

【简历体现】： 能够熟练运用v-model和.sync修饰符，简化封装组件时的父子数据联动

具体话术:

v-model应用于组件，其实是 :value 和 @input 的简写
    <jack v-model="money"></jack> 
    props => value
    this.$emit('input')

.sync应用于组件，其实是 :属性名 和 @update:属性名 的简写
  1. 父组件中，给子组件jack传递参数
    <jack :money="money"></jack> 

  2. 如果希望子组件，可以直接通过触发事件，就更新prop值，就可以添加 sync 修饰符
    <jack :money="money" @update:money="money = $event"></jack> 
    <jack :money.sync="money" :count.sync="count"></jack> 

    this.$emit('update:money', 值)
    this.$emit('update:count', 值)

    .sync 可以多次使用

21-增删改查 #

【面试提问】 简单聊聊xx业务是怎么实现的？

【简历体现】能够熟练实现业务的 CURD （必会）

联想角色列表模块：

  1. C：create 创建
  2. U：update 更新
  3. R：retrieve 读取 - 分页
  4. D：delete 删除

基本的增删改查 （CURD）基本业务 【通用话术】：

  • 首先，这个模块需要发送请求，进行基本的表格渲染，底下是基本的分页
  • 其次，增加和修改一般都可以复用一个弹框，我们公司基本都会封装成组件
  • 那么，如何区分添加和修改呢，通过父传子，定义一个type字段标识做区分 或 通过 form.id
  • 最后，就是删除，就是基本发发请求
